二.String类
	1.字符串缓冲区
		由于字符串在程序中非常常用, 所以Java对其提供了缓冲功能.
		String s1 = "abc";	// 这里先会去缓冲区查找有没有字符串"abc"被缓存, 如果没有则在缓冲区中创建, 之后s1引用缓冲区中的字符串.
		String s2 = "abc";	// 这里也会先去缓冲区找, 在缓冲区找到了字符串"abc", 直接引用之前的, 不会再次创建.
		String s3 = new String("abc");	// 这里用了构造函数, 用构造函数创建字符串不会引用缓冲区中的, 直接在缓冲区外创建, 创建之后再维护缓冲区.
	2.字符串是常量
		字符串在内存中一旦存储, 就不会被改变. 如果一个变量引用一个字符串, 之后将其改变, 不会改变原地址上的值, 而是引用一个新的地址.
	3.字符串相加
		String s = "a" + "b" + "c";	// 这种形式的写法, 其实在编译时直接转为"abc", 没有进行字符串相加.
		字符串的相加是通过StringBuffer类的append()方法实现的, 而通过方法返回的字符串都不维护缓冲区.
	4.字符串的方法
		字符串的所有方法都不会改变字符串的值.
	5.长度问题
		获取数组长度, 是使用数组的length属性, 例如: arr.length
		获取字符串长度, 是使用字符串的length()方法, 例如: s.length()
